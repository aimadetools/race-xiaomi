The Problem: Finding Deprecated Code by Hand

When Claude 4 was retired, developers scrambled to find every reference across their projects. The common approach:

❌ The Manual Way

  • Search for "claude-4-opus" — find 3 hits
  • Search for "claude-sonnet-4" — find 7 more
  • Search for "claude-4-opus-20250615" — miss this one entirely
  • Check config files, environment variables, Docker compose files
  • Remember to check LangChain configs, LlamaIndex settings, Vercel env vars...
  • Miss one reference, debug 410 errors for 2 hours

This takes 15-30 minutes for a small project. For a monorepo with multiple services? Hours.

Framework Auto-Detection

The scanner doesn't just find deprecated model IDs — it identifies which framework you're using so it can show you the right fix:

  • Python SDK — import anthropic → shows updated messages.create() code
  • Node.js SDK — import Anthropic from '@anthropic-ai/sdk' → shows updated constructor
  • LangChain — ChatAnthropic → shows updated model_name parameter
  • LlamaIndex — Claude4 → shows updated model reference
  • Vercel AI SDK — import { anthropic } from '@ai-sdk/anthropic' → shows updated provider
  • CrewAI — Agent configs → shows updated model string
  • Haystack — AnthropicGenerator → shows updated model parameter
  • cURL — Direct API calls → shows updated curl command
