Top Mistakes from Day 1

Not every migration went smoothly. Here are the most common mistakes we saw — and how to avoid them:

🔴 Mistake #1: Forgetting Config Files (40% of tickets)

Developers updated their main code but forgot to check .env files, config.json, and docker-compose.yml. The model ID was hardcoded in 3-4 places they didn't expect. Fix: Run grep -r "claude-4-opus\|claude-4-sonnet" . to find every reference.

🟡 Mistake #2: No Testing After Migration (15% of issues)

Developers swapped the model ID and assumed it worked. But some providers have different output formats or token limits. Fix: Always send a test request after migration. Check the response format matches your expectations.

🟡 Mistake #3: Picking Cheapest Without Testing (10% of rollbacks)

Some teams jumped straight to DeepSeek V4 Flash (cheapest) without testing quality. For complex reasoning tasks, the quality difference was noticeable. Fix: Test with your actual prompts. Use the Replacement Finder to match your use case to the right model.

🟢 Mistake #4: Not Updating SDK Versions

If you're using the Anthropic Python or Node.js SDK, update to the latest version. Older SDKs may have hardcoded model IDs or deprecated API calls. Fix: pip install --upgrade anthropic or npm update @anthropic-ai/sdk.
