How to Estimate Token Usage for Your AI Application

Before you can estimate your AI API costs, you need to estimate your token usage. But tokens are weird — they're not words, they're not characters, and different models tokenize differently. Here's a practical guide to estimating token usage for common AI application patterns.

The Quick Rule of Thumb

For English text, the simplest approximation:

  • 1 word ≈ 1.3 tokens (works for most prose)
  • 1 character ≈ 0.25 tokens (useful for code and structured data)
  • 1 token ≈ 4 characters (the inverse — useful for budgeting)

These rules are approximate but accurate enough for cost estimation. The exact ratio varies by content type:

Content Type Tokens per Word Why
English prose 1.2 - 1.4 Common words are single tokens
Technical/code 1.5 - 2.0 Variable names, operators split into multiple tokens
JSON/structured 1.8 - 2.5 Braces, quotes, keys are separate tokens
Non-English 2.0 - 4.0 Less common in training data, more tokenization needed
Code (Python/JS) 2.0 - 3.0 Indentation, symbols, keywords all tokenize separately

Token Counts by Use Case

Chatbot / Customer Support

Typical Chatbot Request

System prompt (personality + instructions) 200-500 tokens
Conversation history (5 turns) 500-1,500 tokens
User message 20-100 tokens
Assistant response 100-400 tokens
Total per request 800-2,500 tokens

Key insight: The system prompt is a fixed cost on every request. A 500-token system prompt at $2.50/1M input tokens costs $0.00125 per request — it adds up at scale.

RAG (Retrieval-Augmented Generation)

Typical RAG Request

System prompt 200-400 tokens
Retrieved context (2-3 documents) 1,000-5,000 tokens
User question 20-100 tokens
Assistant response 200-600 tokens
Total per request 1,400-6,100 tokens

Key insight: RAG is input-heavy. The retrieved context dominates your input token count. At $2.50/1M tokens (GPT-4o), a 5,000-token context retrieval costs $0.0125 per request.

Code Generation

Typical Code Generation Request

System prompt (coding instructions) 300-600 tokens
User description + existing code 200-2,000 tokens
Generated code response 300-1,500 tokens
Total per request 800-4,100 tokens

Key insight: Code is token-dense. A 100-line Python function is roughly 800-1,200 tokens. Code generation tasks often have high output token counts.

Document Summarization

Typical Summarization Request

System prompt 100-300 tokens
Document content 2,000-10,000 tokens
Summary output 200-500 tokens
Total per request 2,300-10,800 tokens

Key insight: Summarization has the highest input-to-output ratio. You're paying mostly for input tokens. This makes cheap input models (Gemini Flash at $0.10/1M) very attractive.

Monthly Estimation Framework

Here's a simple formula to estimate your monthly token usage:

  1. Estimate tokens per request: Use the breakdowns above for your use case
  2. Multiply by daily requests: How many requests will your app handle per day?
  3. Multiply by 30: Monthly total
  4. Split input/output: Most pricing is different for input vs output tokens

Example: Chatbot at 500 requests/day

Tokens per request (input) 1,500
Tokens per request (output) 300
Daily input tokens 750,000
Daily output tokens 150,000
Monthly input tokens 22.5M
Monthly output tokens 4.5M
GPT-4o cost $101.25/mo
Gemini Flash cost $4.05/mo

Common Estimation Mistakes

  • Forgetting the system prompt: It's sent with every request — a 500-token system prompt adds 15K tokens/day at 30 requests
  • Ignoring conversation history: Chatbots send the full conversation history with each turn — costs grow quadratically
  • Underestimating code tokens: Code is 2-3x more token-dense than prose
  • Forgetting retry/overhead: Add 10-20% buffer for retries, errors, and regeneration
  • Not accounting for growth: If your app grows 3x in 3 months, budget for that

Tools for Exact Token Counting

When you need precision, use these tools:

  • OpenAI Tokenizer: platform.openai.com/tokenizer — visual token counting for GPT models
  • tiktoken (Python): pip install tiktoken — programmatic token counting for OpenAI models
  • Anthropic Tokenizer: Available in the Anthropic SDK — count tokens for Claude models
  • Google AI Studio: Shows token counts in the playground — useful for Gemini models

For quick estimates, paste a sample of your typical input into the OpenAI Tokenizer and note the count. Then multiply by your expected request volume.
