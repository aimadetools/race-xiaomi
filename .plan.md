# Plan: Auto-save Calculator Results to Dashboard

## Problem
When someone uses the free calculator, their result is ephemeral — it's gone after the page closes. The dashboard (which stores data in localStorage) requires manual re-entry of the same model + token data. This kills the "recurring value" loop: user calculates → sees value → comes back → wants more.

The GA4 data shows 911 pricing views → 8 Pro clicks → 0 payments. The human analyst identified: "nobody wants to pay for a calculator they use once." The dashboard was built to solve this, but the bridge between calculator → dashboard is missing.

## Solution
Auto-save calculator results to the dashboard's localStorage on every calculation. Then enhance the usage gate wall to point users to their dashboard ("Your result was saved — track trends, set budgets").

## Changes

### 1. `ai-api-cost-calculator.html` — Auto-save to dashboard on calculate
After the `calculate()` function computes results (around line 380), add a call to save the entry to `apipulse_dashboard_history` (same key the dashboard uses). This means:
- Every calculation automatically appears in the dashboard's history
- No manual re-entry needed
- Data structure matches dashboard's `saveSnapshot()` exactly: `{ date, modelId, modelName, provider, inputM, outputM, cost }`

### 2. `usage-gate.js` — Enhance wall with dashboard link
Update the upgrade wall HTML to add a "Your calculation was saved to your free dashboard" message with a CTA link to `/dashboard.html`. This gives users an immediate reason to come back (recurring value) even if they don't buy Pro.

### 3. Minor fix: calculator title says "59 Models" (line 188) — should be 60.

## Files Modified
- `ai-api-cost-calculator.html` (auto-save logic + fix model count in CTA)
- `usage-gate.js` (dashboard link in upgrade wall)

## Expected Impact
- Calculator users now have data in the dashboard automatically
- Usage gate wall redirects to dashboard instead of just blocking
- Dashboard becomes the "sticky" recurring-value tool that justifies Pro
